# 2026-03-11 工作日志

## 登录流程改造：兼容二次确认弹窗（新设备登录）

### 背景
小红书在新设备登录时会弹出"扫码验证"弹窗（`.captcha-modal-content`），要求用户扫第二个二维码确认身份。无论二维码登录还是手机验证码登录，都可能触发。

### 修改的文件

#### 1. `scripts/xhs/login.py`
- `wait_for_login()` 返回值从 `bool` 改为 `dict[str, object]`，包含 `logged_in`、`captcha_required`、`captcha_data_url` 等字段
- `submit_phone_code()` 返回值从 `bool` 改为 `dict[str, object]`，兼容二次确认弹窗
- 新增 `fetch_captcha_qrcode()` 函数，用于非阻塞场景检测并提取二次确认二维码
- 新增 `_extract_captcha_qrcode()` 内部函数，提取弹窗中的 `.qrcode-container .qrcode-img`
- 移除未使用的 `json` 导入

#### 2. `scripts/cli.py`
- `cmd_login` — 二维码登录阻塞模式：检测到二次弹窗后输出二维码 JSON，继续等待扫码
- `cmd_phone_login` — 交互式手机验证码登录：兼容二次弹窗
- `cmd_verify_code` — 分步验证码提交：兼容二次弹窗
- `cmd_check_login` — 检查登录状态时同时检测二次弹窗
- 新增 `cmd_get_captcha_qrcode` + parser 注册：非阻塞获取二次确认二维码

#### 3. `skills/xhs-auth/SKILL.md`
- 新增"二次确认弹窗说明"章节
- 所有登录方式（A1 有界面二维码、A2 无界面二维码、B 手机验证码）均补充了二次确认处理流程
- 新增"处理二次确认弹窗"独立章节
- 失败处理新增二次确认二维码超时的处理方法

### 二次确认弹窗 DOM 结构（之前调试得到）
```
div.r-captcha-modal.theme-dark
  └── div.captcha-modal-content
        ├── div.captcha-modal__header（标题："扫码验证"）
        └── div
              ├── img.avatar（用户头像，HTTP URL）
              ├── img.qrcode-img（二维码，data URL，176×176）
              └── div.qr-scan-desc（"二维码1分钟失效"）
```
选择器：`CAPTCHA_QRCODE = ".captcha-modal-content .qrcode-container .qrcode-img"`
